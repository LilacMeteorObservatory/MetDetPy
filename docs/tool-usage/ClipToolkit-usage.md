# ClipToolkit User Guide

<center> Language: <b>English</b> | <a href="./ClipToolkit-usage-cn.md">简体中文</a> </center>

`ClipToolkit` (slicing tool) can be used to create multiple video clips from a video or stacked images of those video segments at once. Starting from `MetDetPy v2.2.0`, ClipToolkit has extended the calling method to support more flexible usage and more general scenarios.

## Quick Start

> **Note:** In this document, `[video.mp4]`, `[detection_result.json]`, etc. represent placeholders that need to be replaced with actual file paths.

### Quick Export Clips

ClipToolkit supports quick export of a video segment with specified start and end times or a max-value stacked image:

```sh
# Extract a stacked image from video
# If --save-path is not specified, it will be saved to the current directory by default, and use "video name_start time-end time" as the filename. The input below will generate an image named `video_00_03_00-00_05_00.jpg` to the current directory.
python ClipToolkit.py [video.mp4] --start-time 00:03:00 --end-time 00:05:00

# Specify --save-path to save with a specified name to the specified directory
python ClipToolkit.py [video.mp4] --start-time 00:03:00 --end-time 00:05:00 --save-path ./output/result.jpg
```

By default, ClipToolkit exports images. To export as a video clip, add the `--mode video` parameter:

```sh
# This will export a video clip in .avi format
python ClipToolkit.py [video.mp4]  --mode video --start-time 00:03:00 --end-time 00:05:00
```

### Export Detection Results

> **What is MDRF?**
> MDRF (Meteor Detection Recording Format) is the detection result format used by MetDetPy. This file is automatically generated after running detection with MetDetPy or evaluate. The MDRF file contains complete information such as video information, timestamps of detection results, bounding box coordinates, and confidence scores. See [MDRF Description](../data-format.md#meteor-detection-recording-format-mdrf) for more information.

When you need to batch export detection results, simply provide the MDRF format file as the only positional parameter:

```sh
# Export all segments as video clips to ./output directory
python ClipToolkit.py [detection_result.json] --mode video --save-path ./output
```

When exporting detection results in MDRF format files, you can combine the following parameters to improve export quality:

- **Time Padding**: Add additional time length before and after segments to ensure targets are completely included in the clips.
- **Filter Rules**: Filter out false positives, low-confidence, or too small targets, retaining only high-quality samples.

| Parameter | Description |
|------|------|
| `--enable-filter-rules` | Enable filter rules, see [Filter Rules](#filter-rules). |
| `--padding-before` | Compensation time before the segment start (seconds) |
| `--padding-after` | Compensation time after the segment end (seconds) |
|  `--with-bbox` | Draw bounding boxes on exported images/videos |

The following example will enable filter rules, time padding, and bounding box drawing simultaneously:

```sh
python ClipToolkit.py [detection_result.json] --mode video --with-bbox --enable-filter-rules --padding-before 0.5 --padding-after 0.5 --save-path ./output
```

### Common Parameters

| Scenario | Command |
|------|------|
| Generate video clip | `python ClipToolkit.py [video.mp4] --start-time 00:01:00 --end-time 00:02:00 --mode video` |
| Generate high-quality JPEG image | `python ClipToolkit.py [video.mp4] --start-time 00:01:00 --end-time 00:02:00 --jpg-quality 95` |
| Generate PNG format image (lossless) | `python ClipToolkit.py [video.mp4] --start-time 00:01:00 --end-time 00:02:00 --suffix png` |
| Enable [filter rules](#filter-rules) (export only positive samples) | `python ClipToolkit.py [detection_result.json] --enable-filter-rules` |
| Add time padding before and after segments | `python ClipToolkit.py [detection_result.json] --padding-before 0.5 --padding-after 0.5` |
| Enable [denoising](#denoising-algorithms) | `python ClipToolkit.py [video.mp4] --start-time 00:01:00 --end-time 00:02:00 --denoise mfnr-mix` |
| Generate clips or images with bounding boxes | `python ClipToolkit.py [detection_result.json] --with-bbox` |
| Generate MP4 format video | `python ClipToolkit.py [video.mp4] --start-time 00:01:00 --end-time 00:02:00 --mode video --suffix mp4` |

## Three Input Modes

ClipToolkit supports 3 input modes, determined by the positional arguments passed:

### General Mode

**Use Case:** Need to create multiple image stacks or video clips at once, and configure the name and format of each output.

**Usage:** Pass two positional parameters: `target` (target video file path) and `json` (JSON format string or JSON file path)

**Example:**
```sh
python ClipToolkit.py ./test/20220413Red.mp4 ./test/clip_test.json --mode image --suffix jpg --jpg-quality 60 --save-path ./test
```

**JSON Format Requirements:**
- Must be a JSON array
- Each element contains:
  - Required `"time"` key: Array of two strings in `"HH:MM:SS.ms"` format, representing start time and end time
  - Optional `"filename"` key: Specify filename and extension
  - Optional `"target"` key: Bounding box information, containing `"pt1"`, `"pt2"` and `"preds"`

**JSON Example:**
```json
[
  {
    "time": ["00:03:00.000", "00:05:00.000"],
    "filename": "clip1.jpg"
  },
  {
    "time": ["00:10:00.000", "00:12:00.000"],
    "filename": "clip2.jpg"
  }
]
```

### Simplified Mode

**Use Case:** Only need to process a single image stack or video clip, without building a JSON file.

**Usage:** Pass only one video file as a positional parameter, and specify start/end time, format, etc. through optional parameters.

**Example:**
```sh
python ClipToolkit.py ./test/20220413Red.mp4 --start-time 00:03:00 --end-time 00:05:00 --mode image --save-path ./test/generated_img.jpg
```

**Note:** If `--save-path` contains a file extension (such as `./test/generated_img.jpg`), the program will automatically parse it as the output filename.

### Batch Generation Mode

**Use Case:** Batch generate all corresponding stacked images or video clips for detection results. Mainly used in combination with MetDetPy to batch export detection results.

**Usage:** Specify only one `MDRF` format JSON file as input (this file can be generated by `evaluate` or `MetDetPy`).

**Example:**
```sh
python ClipToolkit.py [video_mdrf.json] --mode video
```

In this mode, the program will generate corresponding images or video clips for each detected target based on the detection results in the MDRF file.

## Complete Parameter Description

```sh
python ClipToolkit.py target [json] [--cfg CFG] [--start-time START_TIME] [--end-time END_TIME]
                              [--mode {image,video}] [--suffix SUFFIX]
                              [--save-path SAVE_PATH]
                              [--jpg-quality JPG_QUALITY] [--png-compressing PNG_COMPRESSING]
                              [--debayer] [--debayer-pattern DEBAYER_PATTERN]
                              [--denoise {mfnr-mix,simple}]
                              [--with-annotation] [--with-bbox]
                              [--enable-filter-rules | --disable-filter-rules]
                              [--padding-before PADDING_BEFORE] [--padding-after PADDING_AFTER]
                              [--debug]
                              [--resource-dir RESOURCE_DIR]
```

### Positional Arguments

| Parameter | Description |
|------|------|
| `target` | Target video file path or MDRF format JSON file path |
| `json` | (Optional) JSON format string or JSON file path, containing time segment information |

### Optional Parameters

#### Basic Parameters

| Parameter | Description | Default Value |
|------|------|--------|
| `--cfg`, `-C` | Configuration file path, manages ClipToolkit's default behavior | `./global/clip_cfg.json` |
| `--mode` | Output sample mode, choose from `{image, video}` | `image` |
| `--suffix` | Output file extension (such as `jpg`, `png`, `mp4`, `avi`) | Image mode `jpg`, video mode `avi` |
| `--save-path` | Output file save path. Can include filename for single segment | Current working directory |
| `--start-time` | Segment start time (milliseconds or `HH:MM:SS.ms` format), valid only in simplified mode | Video start time |
| `--end-time` | Segment end time (milliseconds or `HH:MM:SS.ms` format), valid only in simplified mode | Video end time |

#### Image Quality Parameters

| Parameter | Description | Default Value | Value Range |
|------|------|--------|----------|
| `--jpg-quality` | JPEG image quality | 95 | 0-100 |
| `--png-compressing` | PNG image compression level | 3 | 0-9 |
| `--denoise` | [Denoising algorithm](#denoising-algorithms), choose from `{mfnr-mix, simple}` | Not enabled | - |

#### Annotation Related Parameters

| Parameter | Description | Applicable Modes |
|------|------|----------|
| `--with-annotation` | Also output [Labelme annotation](#labelme-annotation-export) style JSON file | General mode, batch generation mode |
| `--with-bbox` | Draw bounding boxes on exported images/videos | General mode, batch generation mode |

#### Filter Rule Parameters

| Parameter | Description |
|------|------|
| `--enable-filter-rules` | Enable [filter rules](#filter-rules) |
| `--disable-filter-rules` | Disable [filter rules](#filter-rules) |

#### Time Padding Parameters

| Parameter | Description | Default Value |
|------|------|--------|
| `--padding-before` | Time padding's compensation time before segment start (seconds) | `export.clip_padding.before` in configuration file |
| `--padding-after` | Time padding's compensation time after segment end (seconds) | `export.clip_padding.after` in configuration file |

#### Debayer Parameters (Video Mode Only)

| Parameter | Description |
|------|------|
| `--debayer` | For reading raw video, apply Debayer processing to each frame (video mode only) |
| `--debayer-pattern` | Debayer Bayer array pattern, such as `RGGB` or `BGGR` |

#### Debug Parameters

| Parameter | Description |
|------|------|
| `--debug` | Enable debug mode, print more detailed log information |

#### Resource Directory Parameters (configuration when packaged as onefile)

| Parameter | Description |
|------|------|
| `--resource-dir`, `-R` | Resource folder path, should contain `config/`, `weights/`, `resource/` and `global/` subfolders. After specifying, static files will be read from this directory instead of default location |

## Export Format Support

| Output \ Input | RAW | jpg | Timelapse Video | Regular Video |
|-------------|-----|-----|---------|---------|
| Video | x | x | x | ✅ Transcoding, supports MP4/AVI, with audio |
| Video + Bounding Box | x | x | x | ✅ Transcoding, supports MP4/AVI, with audio |
| Image Only | ✅ Copy | ✅ Copy | ✅ Read+Write jpg | ✅ Read+Write jpg |
| Image + Bounding Box | ✅ Read+Write jpg | ✅ Read+Write jpg | ✅ Read+Write jpg | ✅ Read+Write jpg |
| Image + labelme | ✅ Sync Write | ✅ Sync Write | ✅ Sync Write | ✅ Sync Write |

**Legend:**
- x: Not supported/unreasonable requirement
- ✅: Supported

**Notes:**
- Video writer affects available video extensions. See [Configuration File Description](../config-doc.md#video-writer-configuration) for details.
- When attempting to draw bounding boxes on RAW format images, they will be automatically saved as JPG format.

## Advanced Features

### Denoising Algorithms

> **Why need denoising?**
> Videos shot at night typically have more noise, especially at high gain settings. Denoising can significantly improve image quality, making detected targets clearer. Denoising algorithms also attempt to connect possible short lines to generate more aesthetic images.

ClipToolkit supports the following denoising algorithms:

| Algorithm | Parameter Value | Description |
|------|--------|------|
| Multi-frame Noise Reduction Mix | `mfnr-mix` | High-quality denoising, suitable for multi-frame image stacking |
| Simple Denoising | `simple` | Fast denoising, suitable for real-time processing |

**Usage Example:**
```sh
python ClipToolkit.py video.mp4 --start-time 00:01:00 --end-time 00:02:00 --denoise mfnr-mix
```


### Labelme Annotation Export

> **Use Cases:**
> With the `--with-annotation` parameter, you can also generate Labelme style annotation JSON files.
- Prepare training data for deep learning models
- Manual verification of annotation data
- Standardized storage of datasets

**Usage Example:**
```sh
python ClipToolkit.py video.mp4 detections.json --with-annotation --with-bbox
```

### Filter Rules

> **Why need filter rules?**
> Detection result files contain complete results, which will include a large number of false positives, low-confidence or too small targets. Filter rules can automatically filter out these low-quality results, retaining only valid positive samples.

When filter rules are enabled (set in configuration file or specified via command line `--enable-filter-rules`), ClipToolkit will filter targets based on the following rules:

- **Category Filtering:** Exclude built-in negative classes `DROPPED` and `OTHERS`, as well as categories specified in configuration
- **Confidence Filtering:** Filter targets with confidence below threshold
- **Size Filtering:** Filter targets with size relative to image diagonal length ratio that is too small

Filter rules apply simultaneously to image export and video export.

## FAQ

### Q1: How to batch process multiple videos?

**A:** General mode supports specifying multiple segments in one JSON. If you need to process multiple different video files, you can use scripts to loop calling ClipToolkit.

### Q2: Why do I get an error when exporting MP4 format video?

**A:** To export MP4 format videos, you must set `writer` to `FFMpegVideoWriter` in the configuration file. Additionally, you need to download FFMpeg and FFProbe tools, and point the configuration file's `export.ffmpeg_config` to the directory where the above two files are placed. See [Configuration File Description](../config-doc.md#ffmpeg-configuration) for details.

### Q3: Why does my video clip have no audio?

**A:** Ensure you use a VideoWriter that supports audio (such as FFMpegVideoWriter) and configure it correctly in the configuration file. OpenCVVideoWriter does not support audio.

### Q4: How do filter rules affect annotation export?

**A:** [Filter rules](#filter-rules) will first filter out targets that do not meet the criteria, then only perform bounding box drawing and [Labelme annotation](#labelme-annotation-export) export for retained targets.

### Q5: How to enable Debayer processing?

**A:** Use the `--debayer` parameter to enable [Debayer processing](#debayer-parameters-video-mode-only), and use `--debayer-pattern` to specify the Bayer pattern (such as `RGGB`, `BGGR`)

### Q6: Why are colons in the filename replaced with underscores?

**A:** To be compatible with different operating systems, ClipToolkit will automatically replace colons `:` in filenames with underscores `_`. For example: `00:03:00-00:05:00` will become `00_03_00-00_05_00`.
